# TurfCalculator — 项目创意文档

## 一句话定位

面向全球房主的**免费多语言草坪/草皮计算器工具站**，零注册、零广告、即开即用。

---

## 核心创意

### 痛点

房主铺设草坪时需要计算草皮用量、草籽数量、肥料配比、浇水量、土壤需求，但：
- 搜索结果被付费工具和广告淹没
- 现有工具只支持英语
- 单一计算器分散在不同网站，无法一站式完成

### 解决方案

一个**纯静态、零后端**的多功能计算器网站，覆盖草坪铺设全流程，支持 14 种语言。

---

## 产品架构

### 7 大计算工具

| 工具 | 功能 | 核心输入 → 输出 |
|------|------|-----------------|
| 草皮计算器 | 计算草皮用量和费用 | 长×宽 → 草皮卷数、托盘数、总费用 |
| 草坪面积计算器 | 多形状面积测量 | 矩形/圆形/三角形/梯形 → 总面积 |
| 草籽计算器 | 播种量计算 | 面积+草种类型 → 草籽重量(磅) |
| 肥料计算器 | 施肥量计算 | 面积+NPK配比 → 肥料磅数+施肥时机 |
| 浇水计算器 | 灌溉量计算 | 面积+草种 → 每周加仑数+浇水频率 |
| 土壤计算器 | 土壤/覆盖物用量 | 面积+深度 → 立方码/袋数 |
| 面积换算器 | 英制↔公制转换 | sq ft↔sq m、acres↔hectares 等 20 组 |

### 辅助内容页

- **草种类型指南** — 暖季/冷季草种对比、光照/耐旱/耐寒评级
- **草坪养护日历** — 按月份推荐养护操作
- **FAQ** — 首页 4 个高频问答（结构化数据）
- 法律页面 — 隐私政策/条款/免责声明/Cookie政策

### 多语言支持（14 种语言，按 Google 市场规模排序）

英语、中文、西班牙语、阿拉伯语、葡萄牙语、日语、德语、法语、韩语、意大利语、土耳其语、波兰语、荷兰语、瑞典语

→ 14 语言 × 36 页面 = **504 个静态页面**

## 技术架构

### 纯静态站点（零服务器成本）

```
src/                    ← 源码
  build.js              ← Node.js 构建脚本
  templates/*.html      ← 16 个页面模板（含占位符 {{key}}）
  partials/*.html       ← header + footer 组件
  i18n/*.json           ← 14 个语言翻译文件
  js/*.js               ← 9 个计算器逻辑 + app.js
  css/styles.css        ← 全站样式
        ↓ node src/build.js
dist/                   ← 构建产物（504 个 HTML + 静态资源）
```

### 关键设计决策

| 决策 | 原因 |
|------|------|
| 模板占位符 `{{key}}` 替代框架 | 零依赖、构建快、CDN 友好 |
| 构建时 i18n（非运行时） | 每个语言独立 HTML，SEO 完美 |
| 14 个独立 `i18n-{lang}.js` | 按语言加载，不下载多余翻译 |
| Cloudflare Pages 托管 | 免费、全球 CDN、自动构建 |
| Breadcrumb + FAQ JSON-LD | 搜索引擎富文本摘要 |
| `hreflang` 标签 | Google 正确索引多语言页面 |
| RTL 支持（阿拉伯语） | `dir="rtl"` + CSS 逻辑属性 |

### 构建流程

1. 读取模板 + 翻译文件
2. 替换 `{{key}}` 占位符为翻译文本
3. 注入 SEO meta（og:title、canonical、hreflang）
4. 注入结构化数据（BreadcrumbList、FAQPage）
5. 生成 sitemap.xml + robots.txt
6. 复制 CSS/JS + 生成 i18n JS 文件
7. 输出到 `dist/`

## SEO 策略

### 页面结构

每个计算器页面 = 独立 URL + 独立翻译：
```
/en/sod-calculator.html
/zh/草皮计算器.html
/es/calculadora-de-cesped.html
/ar/حاسبة-العشب.html
```

### 关键 SEO 元素

- 每页独立 `<title>` + `<meta description>` + `<h1>`
- `<link rel="alternate" hreflang="xx">` 全语言交叉引用
- `<link rel="canonical">` 规范化 URL
- `<meta name="robots" content="index, follow">`
- BreadcrumbList + FAQPage 结构化数据
- sitemap.xml（36 个页面入口 × hreflang）
